# Debian 운영체제에 대한 기초 학습

## Linux와 Debian
많은 초심자들이 **리눅스(Linux)** 와 **데비안(Debian)** 의 차이를 혼동한다. 데비안은 리누스 토르발즈(https://github.com/torvalds/linux) 의 리눅스 커널을 기반으로 한 **리눅스 배포판(distribution)** 이다. 이런 배포판들은 데비안 외에도 centOS, Rocky, Arch 등 다양한 종류가 존재하며 이들은 리눅스 커널과 함께 다양한 소프트웨어 패키지, 라이브러리, 도구들을 포함하여 완전한 운영체제를 구성한다. 따라서 Linux는 운영체제의 핵심인 커널을 의미하며, 데비안은 그 커널을 기반으로 한 운영체제 전체를 의미한다.

## Debian과 Ubuntu
**데비안(Debian)** 과 **우분투(Ubuntu)** 는 모두 동일한 리눅스 커널을 기반으로 한 리눅스 배포판의 일종이다. 통상 우분투를 일컫어 데비안의 파생이라고 부른다. 이유는 우분투가 데비안의 패키지와 소프트웨어를 기반으로 개발되었기 때문이다. 그러나 두 배포판은 몇 가지 중요한 차이점이 있다.

대표적으로는 커널 버전의 차이가 그러하다. **데비안은 안정성을 중시** 하여 상대적으로 오래된 커널 버전을 사용하는 반면, **우분투는 최신 기능을 반영** 하기 위해 더 자주 커널을 업데이트한다. 이로 인해 우분투는 최신 하드웨어에 대한 지원이 더 뛰어난 반면, 데비안은 안정성과 보안성을 중시하는 경향이 있다. 이러한 연유로 서버 환경에서는 데비안이, 데스크탑 환경에서는 우분투가 더 선호되는 경향이 있다. 만약 당신이 리눅스를 처음 사용해본다면 우분투를 선택하는 것이 더 나은 경험이 될 수 있다.

## Package
**패키지(package)** 란 소프트웨어의 배포 및 관리를 위한 단위로, 프로그램과 그에 필요한 라이브러리, 설정 파일 등을 포함하는 **아카이브 파일** 이다. 패키지는 특정 형식으로 묶여 있으며, 이를 통해 패키지 매니저가 소프트웨어를 쉽게 설치, 업데이트, 제거할 수 있도록 돕는다.

데비안의 경우 패키지의 타입으로 **.deb** 을 사용한다. 이는 데비안 패키지 형식(Debian Package Format)으로, 데비안 및 그 파생 배포판에서 널리 사용된다. 또한 패키지 매니저의 경우는 **APT(Advanced Package Tool)** 를 사용한다.

필자는 Linux 운영체제를 주로 Ubuntu 배포판을 사용해왔다. 또한 가상머신에서 무인 설치 옵션(unattended installation)을 주로 활용하였다. 그러므로 명령줄 인터페이스(command line interface, CLI) 기반의 최소한의 패키지로만 구성된 데비안을 처음 접했을 때, 다소 어려움을 느꼈다. 특히 과제에서 요구하는 기준을 달성하기 위해 특정 명령, 혹은 프로그램을 실행하고자 하는데, command not found와 같은 오류 메시지를 자주 접하게 되었다. 이런 문제를 해결하기 위해 가장 먼저 시도한 것은 패키지 설치였다. 데비안에서는 기본적으로 제공되지 않는 패키지가 많기 때문에, 필요한 패키지를 수동으로 설치해야 했다.

APT는 대표적으로 다음과 같은 명령어를 제공한다.

**`apt`, `apt-get`, `aptitude`**

위 세 명령은 기본적으로 혼용이 가능하다. 하지만 그럼에도 불구하고 각각의 차이가 존재한다.

APT는

**"APT provides a high-level commandline interface for the package management system."**

라는 메뉴얼 설명이 있듯이, 패키지 관리 시스템을 위한 고급 명령줄 인터페이스를 제공한다. 따라서 `apt` 명령어는 `apt-get`과 `apt-cache`의 기능을 통합하여 보다 사용자 친화적인 인터페이스를 제공한다.

반면 `apt-get`은 

**"apt-get is the command-line tool for handling packages, and may be considered the user's "back-end" to other tools using the APT library."** 

이라는 메뉴얼 설명에서 볼 수 있듯, `apt` 보다 낮은 레벨에서 패키지를 다루는 도구이다.

마지막으로 `aptitude`는 

**"aptitude command comes with rich features such as its enhanced package resolve"**

라는 데비안 메뉴얼 설명에서 볼 수 있듯, 보다 고급스러운 패키지 관리 기능을 제공하는 도구이다. 하지만 이런 풍부한 기능이 종종 몇 몇 오류를 발생시키기도 한다.

이와 같은 이유로 일반 사용자는 대부분 데비안에서 패지키 관리 도구로 `apt`만을 사용해도 큰 문제가 발생하지 않는다. 여담으로 `pip`는 Python 패키지 관리를 위한 도구로, 일부 APT와 유사한 기능을 제공한다. 하지만, pip는 Python 패키지 전용으로 설계되었기 때문에 시스템 전반에 걸친 패키지 관리를 필요로 하는 경우 APT를 사용하는 것이 더 적합하다.

## Access Control
보안과 관련하여 **AAA 모델** 이란 것이 존재한다. 이 모델에서 각 알파벳이 나타내는 것은 각각 **인증(authentication)**, **인가(authorization)**, **계정관리(Accounting)** 이다. 데비안에서 **접근제어(access control)** 는 이 중 인가와 관련된 작업을 수행한다. 그렇다면 접근제어란 무엇인가, 위키피디아에 따르면 다음과 같이 정의된다.

**"사람이나 프로세스가 시스템이나 파일에 읽기, 쓰기, 실행 등의 접근 여부를 허가하거나 거부하는 기능"**

이러한 접근제어는 다시 크게 2개의 파트로 구분된다.

### 임의접근제어(Discretionary Access Control, DAC)

임의접근제어는 다음과 같이 정의된다.

**"주체의 신원 및 소속 그룹에 기반하여 객체에 대한 접근을 제한하는 수단"**

이걸 데비안에서 특정 파일의 소유주 혹은 그룹에 데한 읽기, 쓰기, 실행 권환을 설정하는 것과 연관되어 생각할 수 있다. 예를 들어 `ls -l [file_name]` 과 같은 명령으로 어떤 파일에 대한 권환을 출력하면 다음과 같은 결과가 나온다.

```bash
-rw-r--r-- 1 user group 0 Jan 1 00:00 file.txt
```
여기서 첫째 열 그룹에 나타난 "-rw-r--r--"가 파일의 소유주, 소유그룹 그리고 다른 사용자들에 대한 읽기, 쓰기, 실행 권환 등을 나타낸다. 이를 변경하려면 `chmod`와 같은 명령을 사용할 수 있다.

### 강제접근제어(Mandatory Access Control, MAC)

일단, 여기서 MAC은 MAC(Media Access Control) 주소가 아니다. 이에 대한 내용은 다음을 참고하라. [link](https://ko.wikipedia.org/wiki/MAC_%EC%A3%BC%EC%86%8C)

강제접근제어는 **규칙기반접근제어(Rule Based Access Control, RBAC)**라고도 불린다. 운영체제와 같은 시스템 관리자만 접근 권한을 설정할 수 있으며, 사용자가 직접 권한을 변경할 수는 없다.

데비안에서는 **AppArmor**와 **SELinux**와 같은 MAC 구현체를 제공한다.

AppArmor는 경로기반 접근제어를 제공하며, 특정 프로그램이 접근할 수 있는 파일 및 리소스를 제한한다. 반면, SELinux는 메타데이터를 기반으로 한 보다 포괄적인 보안 정책을 제공하며, 시스템 전체에 대한 접근 제어를 설정할 수 있다.

일반적으로 SELinux는 더 복잡한 설정을 요구하지만, 그만큼 강력한 보안 기능을 제공한다.

**정정 및 보충** 데비안에서는 AppArmor가 기본적으로 활성화되어 있지만, SELinux는 기본적으로 비활성화되어 있다. 따라서 데비안에서 MAC을 활용하려면 AppArmor를 사용하는 것이 일반적이다.

## fire wall
**방화벽(fire wall)** 이란 보안 규칙에 기반하여 네트워크 트래픽을 모니터링하고 제어하는 시스템이다. 방화벽은 허용된 트래픽과 차단된 트래픽을 구분하여, 외부의 위협으로부터 내부 네트워크를 보호하는 역할을 한다.

데비안에서는 **UFW**(Uncomplicated Firewall)라는 방화벽 관리 도구를 제공한다. UFW는 복잡한 iptables 명령어 대신 간단한 명령어로 방화벽 규칙을 설정할 수 있도록 도와준다.

구체적으로는 OSI 7계층에 L3/L4에 해당하는 네트워크 및 전송 계층에서 동작하는 방화벽으로 IP 주소(L3)와 전송 프로토콜(L4) 그리고 포트 넘버에 기반하여 패킷의 입출력을 제어한다. 만약 이 문단에서 나오는 용어가 생소하다면 다음을 참고하라. [link](https://namu.wiki/w/OSI%20%EB%AA%A8%ED%98%95)

## volume and partition
_**주의** 이 부분에 대한 내용은 충분히 공신력 있는 자료를 참고하지 못했다._<br>
**볼륨(volume)** 과 **파티션(partition)** 은 일반적으로 혼동되어 사용되곤 하나 둘은 서로 다른 개념이다. 이 둘은 시스템에서 물리적 저장장치인 **디스크(disk)를 어떻게 사용할 것 인가 결정하는 요소** 이다. 먼저 디스크는 적어도 하나의 파티션으로 포맷된다. 이 때, **파티션은 물리적 저장 장치에 대한 논리적 구획** 으로, 디스크의 전체를 사용할 수 도 혹은 디스크의 일부를 사용할 수 도 있으며, 하나의 파티션은 반드시 하나의 파일 시스템 _(※파일 시스템에 대하여서는 다음을 참고하라. [link](https://ko.wikipedia.org/wiki/%ED%8C%8C%EC%9D%BC_%EC%8B%9C%EC%8A%A4%ED%85%9C))_ 으로 포맷되어야 한다. 

포맷된 **파티션은 다시 볼륨으로 추상화** 되며, 이 때 볼륨은 반드시 단일 종류의 파일 시스템을 가진 파티션으로만 구성되어야 한다. 이는 볼륨이 운영체제와 직접적으로 소통하며, 파일을 관리하는 단위가 되기 때문이다. _(※이는 다르게 말하면, 하나의 모든 파티션이 하나의 파일 시스템으로만 구성되어있으며, 볼륨이 여러 개의 파티션에 걸쳐져서 생성될 수 도 있다는 뜻이다.)_ 

데비안에서는 이와 관련하여 **논리 볼륨 매니저(Logical Volume Manager, LVM)** 을 제공한다. LVM은 생성된 파티션을 거대한 볼륨 그룹으로 묶고, 사용자에 필요에 따라 논리적 볼륨을 생성하여 관리하는 툴이다. 추상화를 거치었기에 LVM을 사용하지 않은 디스크 사용과 비교하여 공간의 관리가 매우 수월하게 이루어질 수 있다.

## LUKS
**LUKS(Linux Unified Key Setup)** 는 리눅스에서 **디스크 암호화를 위한 표준 포맷** 이다. LUKS는 전체 디스크 또는 특정 파티션을 암호화하여 데이터의 기밀성을 보호하는 데 사용된다. LUKS는 여러 키 슬롯을 지원하여, 사용자가 암호를 잊어버렸을 때에도 데이터에 접근할 수 있는 방법을 제공한다.

데비안에서는 LUKS를 사용하여 쉽게 디스크 암호화를 설정할 수 있다. 이를 통해 민감한 데이터를 안전하게 보호하고, 무단 접근으로부터 시스템을 방어할 수 있다. LUKS를 사용하여 **LVM에 암호화 기능을 더 할 수 도 있다.**


## PAM
**PAM(Pluggable Authentication Modules)** 은 리눅스 시스템에서 인증(authentication)과 관련된 작업을 모듈화하여 처리하는 프레임워크이다. PAM은 다양한 인증 방법을 지원하며, 시스템 관리자가 필요에 따라 인증 정책을 쉽게 구성할 수 있도록 도와준다.
