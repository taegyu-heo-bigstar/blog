# Debian 운영체제에 대한 기초 학습
	본 문서에서는 Debian 운영체제의 기본 개념과 특징에 대해 학습한 내용을 정리합니다.
	본 문서가 작성된 배경에는 42gyeongsan의 Born2beroot 프로젝트가 있습니다.

## Debian 운영체제에 대한 기본 개념
	Debian 현대에 아주 많이 쓰이는 리눅스 배포판인 우분투의 기반이 되는 운영체제이다. 높은 안정성과 보안성을 제공하며, 다양한 소프트웨어 패키지를 지원한다. 또한, 커뮤니티 기반으로 개발되고 유지보수되며, 자유 소프트웨어 철학을 준수한다.

## 패키지 관리 도구
	필자는 Linux 운영체제를 주로 Ubuntu 배포판을 사용해왔다. 또한 가상머신에서 무인 설치 옵션(unattended installation)을 주로 활용하였다. 그러므로 cli 기반의 최소한한의 패키지로만 구성된 Debian을 처음 접했을 때, 다소 어려움을 느꼈다. 특히 과제에서 요구하는 기준을 달성하기 위해 특정 명령, 혹은 프로그램을 실행하고자 하는데, command not found와 같은 오류 메시지를 자주 접하게 되었다. 이런 문제를 해결하기 위해 가장 먼저 시도한 것은 패키지 설치였다. Debian에서는 기본적으로 제공되지 않는 패키지가 많기 때문에, 필요한 패키지를 수동으로 설치해야 했다.

	먼저, **패키지**란 무엇인자 살펴보자.

	패키지란 위키피디아에서 다음과 같이 정의된다.
	"instance of this(package format) type of archive"

	그리고 패키지 포맷은 다시 다음과 같이 정의된다.
	"Package format is a type of archive containing computer programs and additional metadata needed by package managers"

	즉 패키지란 패키지 매니저가 필요로 하는 메타데이터와 함께 컴퓨터 프로그램을 포함하는 아카이브의 인스턴스이다. 패키지는 우리가 단순히 제작 및 사용하는 a.out과 같은 파일과는 달리 패키지 매니저가 인식할 수 있는 형식으로 묶여 있다. 이는 종속성의 관리와 함께 소프트웨어의 설치, 업그레이드, 제거를 용이하게 한다.

	그러면 패키지 매니저가 뭐하는 녀석일까. 다시 한 번 위키피디아를 보자.
	"A package manager or package management system is software that supports installing, upgrading, configuring, and removing software for the host system in a consistent manner."

	



	apt, apt-get, aptitude
