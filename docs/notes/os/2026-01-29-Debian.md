# Debian 운영체제에 대한 기초 학습
본 문서에서는 Debian 운영체제의 기본 개념과 특징에 대해 학습한 내용을 정리합니다.
본 문서가 작성된 배경에는 42gyeongsan의 Born2beroot 프로젝트가 있습니다.

## Debian 운영체제에 대한 기본 개념
Debian 현대에 아주 많이 쓰이는 리눅스 배포판인 우분투의 기반이 되는 운영체제이다. 높은 안정성과 보안성을 제공하며, 다양한 소프트웨어 패키지를 지원한다. 또한, 커뮤니티 기반으로 개발되고 유지보수되며, 자유 소프트웨어 철학을 준수한다.

## 패키지 관리 도구
필자는 Linux 운영체제를 주로 Ubuntu 배포판을 사용해왔다. 또한 가상머신에서 무인 설치 옵션(unattended installation)을 주로 활용하였다. 그러므로 cli 기반의 최소한의 패키지로만 구성된 Debian을 처음 접했을 때, 다소 어려움을 느꼈다. 특히 과제에서 요구하는 기준을 달성하기 위해 특정 명령, 혹은 프로그램을 실행하고자 하는데, command not found와 같은 오류 메시지를 자주 접하게 되었다. 이런 문제를 해결하기 위해 가장 먼저 시도한 것은 패키지 설치였다. Debian에서는 기본적으로 제공되지 않는 패키지가 많기 때문에, 필요한 패키지를 수동으로 설치해야 했다.

먼저, **패키지**란 무엇인자 살펴보자.

패키지란 위키피디아에서 다음과 같이 정의된다.

**"instance of this(package format) type of archive"**

그리고 패키지 포맷은 다시 다음과 같이 정의된다.

**"Package format is a type of archive containing computer programs and additional metadata needed by package managers"**

즉 패키지란 패키지 매니저가 필요로 하는 메타데이터와 함께 컴퓨터 프로그램을 포함하는 아카이브의 인스턴스이다. 패키지는 우리가 단순히 제작 및 사용하는 a.out과 같은 파일과는 달리 패키지 매니저가 인식할 수 있는 형식으로 묶여 있다. 이는 종속성의 관리와 함께 소프트웨어의 설치, 업그레이드, 제거를 용이하게 한다.

이런 패키지를 관리하는 도구가 바로 **패키지 매니저**이다. 패키지 매니저는 소프트웨어 패키지의 설치, 업그레이드, 제거 및 관리를 자동화하는 도구이다. 패키지 매니저는 패키지의 종속성을 해결하고, 소프트웨어의 버전을 관리하며, 시스템의 일관성을 유지하는 데 도움을 준다.

대표적인 패키지 매니저로는 **APT**(Advanced Package Tool)가 있다. APT는 Debian 및 Ubuntu 기반 시스템에서 소프트웨어 패키지를 관리하는 데 사용된다. APT는 패키지의 설치, 업그레이드, 제거를 간편하게 수행할 수 있는 명령어 인터페이스를 제공한다.

APT는 대표적으로 다음과 같은 명령어를 제공한다.

`apt`, `apt-get`, `aptitude`

위 세 명령은 기본적으로 혼용이 가능하다. 하지만 그럼에도 불구하고 각각의 차이가 존재한다.

apt는

**"apt provides a high-level commandline interface for the package management system."**

라는 메뉴얼 설명이 있듯이, 패키지 관리 시스템을 위한 고급 명령줄 인터페이스를 제공한다. 따라서 `apt` 명령어는 `apt-get`과 `apt-cache`의 기능을 통합하여 보다 사용자 친화적인 인터페이스를 제공한다.

반면 apt-get은 "apt-get is the command-line tool for handling packages, and may be considered the user's "back-end" to other tools using the APT library." 이라는 메뉴얼 설명에서 볼 수 있듯, apt 보다 낮은 레벨에서 패키지를 다루는 도구이다.

마지막으로 aptitude는 "aptitude command comes with rich features such as its enhanced package resolve"라는 debian 메뉴얼 설명에서 볼 수 있듯, 보다 고급스러운 패키지 관리 기능을 제공하는 도구이다. 하지만 이런 풍부한 기능이 종종 몇 몇 오류를 발생시키기도 한다.

이와 같은 이유로 일반 사용자는 대부분 debain에서 패지키 관리 도구로 apt만을 사용해도 큰 문제가 발생하지 않는다. 여담으로 pip는 Python 패키지 관리를 위한 도구로, APT와 유사한 기능을 제공한다. 하지만, pip는 Python 패키지 전용으로 설계되었기 때문에 시스템 전반에 걸친 패키지 관리를 필요로 하는 경우 APT를 사용하는 것이 더 적합하다.
