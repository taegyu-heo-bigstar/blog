본 문서에서는 Debian 운영체제의 기본 개념과 특징에 대해 학습한 내용을 정리합니다.<br>

## Debian 운영체제에 대한 기본 개념
Debian 현대에 아주 많이 쓰이는 리눅스 배포판인 우분투의 기반이 되는 운영체제이다. 높은 안정성과 보안성을 제공하며, 다양한 소프트웨어 패키지를 지원한다. 또한, 커뮤니티 기반으로 개발되고 유지보수되며, 자유 소프트웨어 철학을 준수한다.

## 패키지 관리 도구
필자는 Linux 운영체제를 주로 Ubuntu 배포판을 사용해왔다. 또한 가상머신에서 무인 설치 옵션(unattended installation)을 주로 활용하였다. 그러므로 cli 기반의 최소한의 패키지로만 구성된 Debian을 처음 접했을 때, 다소 어려움을 느꼈다. 특히 과제에서 요구하는 기준을 달성하기 위해 특정 명령, 혹은 프로그램을 실행하고자 하는데, command not found와 같은 오류 메시지를 자주 접하게 되었다. 이런 문제를 해결하기 위해 가장 먼저 시도한 것은 패키지 설치였다. Debian에서는 기본적으로 제공되지 않는 패키지가 많기 때문에, 필요한 패키지를 수동으로 설치해야 했다.

먼저, **패키지**란 무엇인지 살펴보자.

패키지란 위키피디아에서 다음과 같이 정의된다.

**"instance of this(package format) type of archive"**

그리고 패키지 포맷은 다시 다음과 같이 정의된다.

**"Package format is a type of archive containing computer programs and additional metadata needed by package managers"**

즉 패키지란 패키지 매니저가 필요로 하는 메타데이터와 함께 컴퓨터 프로그램을 포함하는 아카이브의 인스턴스이다. 패키지는 우리가 단순히 제작 및 사용하는 a.out과 같은 파일과는 달리 패키지 매니저가 인식할 수 있는 형식으로 묶여 있다. 이는 종속성의 관리와 함께 소프트웨어의 설치, 업그레이드, 제거를 용이하게 한다.

이런 패키지는 기본적으로 원격 저장소에 존재하며, 패키지 매니저를 통해 로컬 시스템에 설치된다. 그러므로 패키지 매니저의 역할이 매우 중요하다 할 수 있다. 이와 같은 패키지 매니저로는 대표적으로 **APT**(Advanced Package Tool)가 있다. APT는 Debian 및 Ubuntu 기반 시스템에서 소프트웨어 패키지를 관리하는 데 사용된다. APT는 패키지의 설치, 업그레이드, 제거를 간편하게 수행할 수 있는 명령어 인터페이스를 제공한다.

APT는 대표적으로 다음과 같은 명령어를 제공한다.

`apt`, `apt-get`, `aptitude`

위 세 명령은 기본적으로 혼용이 가능하다. 하지만 그럼에도 불구하고 각각의 차이가 존재한다.

APT는

**"APT provides a high-level commandline interface for the package management system."**

라는 메뉴얼 설명이 있듯이, 패키지 관리 시스템을 위한 고급 명령줄 인터페이스를 제공한다. 따라서 `apt` 명령어는 `apt-get`과 `apt-cache`의 기능을 통합하여 보다 사용자 친화적인 인터페이스를 제공한다.

반면 `apt-get`은 

**"apt-get is the command-line tool for handling packages, and may be considered the user's "back-end" to other tools using the APT library."** 

이라는 메뉴얼 설명에서 볼 수 있듯, `apt` 보다 낮은 레벨에서 패키지를 다루는 도구이다.

마지막으로 `aptitude`는 

**"aptitude command comes with rich features such as its enhanced package resolve"**

라는 debian 메뉴얼 설명에서 볼 수 있듯, 보다 고급스러운 패키지 관리 기능을 제공하는 도구이다. 하지만 이런 풍부한 기능이 종종 몇 몇 오류를 발생시키기도 한다.

이와 같은 이유로 일반 사용자는 대부분 debian에서 패지키 관리 도구로 `apt`만을 사용해도 큰 문제가 발생하지 않는다. 여담으로 `pip`는 Python 패키지 관리를 위한 도구로, 일부 APT와 유사한 기능을 제공한다. 하지만, pip는 Python 패키지 전용으로 설계되었기 때문에 시스템 전반에 걸친 패키지 관리를 필요로 하는 경우 APT를 사용하는 것이 더 적합하다.

## 접근제어(Access Control)
접근제어란 무엇인가, 위키피디아에 따르면 다음과 같이 정의된다.

**"사람이나 프로세스가 시스템이나 파일에 읽기, 쓰기, 실행 등의 접근 여부를 허가하거나 거부하는 기능"**

이러한 접근제어는 다시 크게 2개의 파트로 구분된다.

### 임의접근제어(Discretionary Access Control, DAC)

임의접근제어는 다음과 같이 정의된다.

**"주체의 신원 및 소속 그룹에 기반하여 객체에 대한 접근을 제한하는 수단"**

이걸 Debian에서 특정 파일의 소유주 혹은 그룹에 데한 읽기, 쓰기, 실행 권환을 설정하는 것과 연관되어 생각할 수 있다. 예를 들어 `ls -l [file_name]` 과 같은 명령으로 어떤 파일에 대한 권환을 출력하면 다음과 같은 결과가 나온다.

```bash
-rw-r--r-- 1 user group 0 Jan 1 00:00 file.txt
```
여기서 첫째 열 그룹에 나타난 "-rw-r--r--"가 파일의 소유주, 소유그룹 그리고 다른 사용자들에 대한 읽기, 쓰기, 실행 권환 등을 나타낸다. 이를 변경하려면 `chmod`와 같은 명령을 사용할 수 있다.

### 강제접근제어(Mandatory Access Control, MAC)

일단, 여기서 MAC은 MAC(Media Access Control) 주소가 아니다. 이에 대한 내용은 다음을 참고하라. [link](https://ko.wikipedia.org/wiki/MAC_%EC%A3%BC%EC%86%8C)

강제접근제어는 **규칙기반접근제어**(Rule Based Access Control, RBAC)라고도 불린다. 운영체제와 같은 시스템 관리자만 접근 권한을 설정할 수 있으며, 사용자가 직접 권한을 변경할 수는 없다.

Debian에서는 **AppArmor**와 **SELinux**와 같은 MAC 구현체를 제공한다.

AppArmor는 경로기반 접근제어를 제공하며, 특정 프로그램이 접근할 수 있는 파일 및 리소스를 제한한다. 반면, SELinux는 메타데이터를 기반으로 한 보다 포괄적인 보안 정책을 제공하며, 시스템 전체에 대한 접근 제어를 설정할 수 있다.

일반적으로 SELinux는 더 복잡한 설정을 요구하지만, 그만큼 강력한 보안 기능을 제공한다.

## 방화벽(fire wall)
방화벽이란 보안 규칙에 기반하여 네트워크 트래픽을 모니터링하고 제어하는 시스템이다. 방화벽은 허용된 트래픽과 차단된 트래픽을 구분하여, 외부의 위협으로부터 내부 네트워크를 보호하는 역할을 한다.

Debian에서는 **UFW**(Uncomplicated Firewall)라는 방화벽 관리 도구를 제공한다. UFW는 복잡한 iptables 명령어 대신 간단한 명령어로 방화벽 규칙을 설정할 수 있도록 도와준다.

구체적으로는 OSI 7계층에 L3/L4에 해당하는 네트워크 및 전송 계층에서 동작하는 방화벽으로 IP 주소(L3)와 전송 프로토콜(L4) 그리고 포트 넘버에 기반하여 패킷의 입출력을 제어한다. 만약 이 문단에서 나오는 용어가 생소하다면 다음을 참고하라. [link](https://namu.wiki/w/OSI%20%EB%AA%A8%ED%98%95)

## 파티션
파티션이란 물리적인 저장 장치를 논리적으로 나누는 것을 의미한다. 각 파티션은 독립적인 파일 시스템을 가질 수 있으며, 운영체제는 이를 별도의 드라이브로 인식한다.

**LVM**
